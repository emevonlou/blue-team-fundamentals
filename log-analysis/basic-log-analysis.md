# Basic Log Analysis Exercise

## Objective
Understand how to identify suspicious activity through system logs.

## Scenario
A Linux system shows multiple failed login attempts in a short period of time.

## What to Look For
- Repeated authentication failures
- Source IP patterns
- Time correlation between events

## Questions
- Could this indicate a brute-force attempt?
- What logs would you prioritize?
- What would be the next response step?

## Notes
This exercise focuses on reasoning, not tools.
